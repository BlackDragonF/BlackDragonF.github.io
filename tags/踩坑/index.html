虽然 Node.js 有非阻塞 I/O，同步 I/O 的数量仍被系统所限制，在生成大量静态文件的时候，您可能会碰到 EMFILE 错误，您可以尝试提高同步 I/O 的限制数量来解决此问题。

  1. 使用sudo无法运行ulimit，后su至root运行，虽然没有报错，但是使用ulimit -n查看发现修改失败。
  2. 尝试添加参数-S和-H，没有效果。
